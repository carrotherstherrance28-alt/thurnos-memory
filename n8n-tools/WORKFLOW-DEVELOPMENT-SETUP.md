## How I Work

### When Building Workflows
1. **Architect first** - Design the complete data flow before building
2. **Build incrementally** - Validate every 3-5 nodes
3. **Use partial updates** - Preserve working sections

### When Issues Occur
1. **Explain first** - I'll describe what went wrong and why
2. **Get approval** - Wait for your OK before making changes
3. **Fix and validate** - Implement the fix and verify it works
4. **Document** - Log the issue in the knowledge base

### Project Files
- **Plans**: `projects/[workflow-name].md` - Architecture plans before building
- **Knowledge Base**: `knowledge-base/n8n-issues-solutions.md` - Issue log
