Before building any new skill, workflow, automation, or tool, always search first.

**Why:** Saves time, avoids reinventing what already exists, and compounds faster by adapting proven work.

**How to apply:**

1. **Search GitHub first** — Priority sources:
   - `n8n-io/n8n-workflows`
   - awesome-mcp-servers repos
   - Topic tags: `n8n`, `mcp`, `ai-agent`, `automation`

2. **Report findings** — Short summary of what exists and % match to the goal

3. **Decision rule:**
   - 70%+ match → Adapt existing resource, document changes
   - Below 70% → Build from scratch, document architecture

4. **Never build blind** — Confirm with Therrance before starting a full build if a close match was found

5. **Log everything** — Record what was searched, what was found, what was built/adapted
